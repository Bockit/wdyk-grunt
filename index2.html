Evolution

Over time our build processes has evolved.

  • New technologies
  • New requirements
  • Laziness

Javascript Build

We used AMD for a long time

r.js -o build.js
r.js -o build-ie.js

And recently swapped to browserify

browserify main.js -t coffeeify -o bundle.js

SCSS and Stylus

For a long time we used SCSS with compass

cd sass
compass watch

and recently swapped to Stylus.

stylus -w -o . src/styl/main.styl

Pushing to production

Changes almost project to project.

  • Uploading manually to S3
  • Uploading manually to FTP
  • git pull on server
  • git aws.push, git push heroku master, etc.

What's the problem

There is overhead per project for accomplishing similar tasks

  • Building

What is Grunt?

Self-described task runner.

Command line tool

You define tasks that you want to automate.

Use the command line tool to perform those tasks.

grunt watch
grunt compass:dev
grunt browserify:dev
grunt dev
grunt test

Example tasks are:

  • compile stylus
  • compile coffeescript
  • compile jade
  • compress images
  • clean up files
  • watching files
  • uploading to s3
  • set environment variables
  • testing
  • minify javascript
  • prompt user for input and write to a config file
  • ... anything!

Installing

http://gruntjs.com has an excellent guide on the specifics of how to install.

The long and short of it is:

npm install -g grunt-cli
cd some/project/with/Gruntfile.js/and/package.json/
# Gruntfile.coffee also works
# Make sure grunt (not grunt-cli) is in your package dev dependencies
npm install
grunt -h

Using Grunt

  • Make a file Gruntfile.js (or .coffee) in your package root.
  • Add a package.json if you haven't already.
  • npm install grunt --save-dev
  • Add the following code to your gruntfile:
module.exports = function(grunt) {
    grunt.initConfig({})
    
    grunt.registerTask('my-first-task', function() {
        console.log('Hello, world!')
    })
}

$ grunt my-first-task

Using Grunt continued

  • We want to do more than echo out 'Hello, world!'
  • There are hundreds of pre-written tasks that we just have to configure.
    • module.exports = function(grunt) {
    grunt.initConfig({
        jade: {
            compile: {
                files: {
                    'index.html': 'index.jade'
                }
            }
        }
    })
    
    grunt.loadNpmTasks('grunt-contrib-jade');
}

    $ grunt jade:compile
